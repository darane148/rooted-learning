# Week 1 - Day 1 - Networking Fundamentals

## 🧠 What is a Network?

A network is a collection of interconnected devices that communicate with each other to share data and resources.

Examples:
- Internet
- Local Wi-Fi network
- Office networks

---

## ⚙️ How a Network Works?

- Devices are connected using wired or wireless mediums  
- Each device has an IP address for identification  
- Data is transmitted in the form of packets  
- Routers and switches help forward data to the correct destination  

---

## 🌐 Types of Networks

- **LAN (Local Area Network)** → small area (home, office)  
- **WAN (Wide Area Network)** → large area (internet)  
- **MAN (Metropolitan Area Network)** → city-level network  

---

## 🖥️ Network Devices

- **Router** → connects networks and routes traffic  
- **Switch** → connects devices within a network  
- **Hub** → basic device (broadcasts data to all ports)  

---

## 🆔 Addressing in Networks

### 🌍 IP Address
- Unique identifier for devices  

#### IPv4 vs IPv6:
- IPv4 → 192.168.1.1  
- IPv6 → longer format (more secure and scalable)  

### 🔗 MAC Address
- Physical hardware address (unique per device)  

---

## 🌐 DNS (Domain Name System)

- Converts domain names → IP address  
- Example: google.com → IP  

💡 Without DNS, we must remember IP addresses manually  

---

## 📡 DHCP (Dynamic Host Configuration Protocol)

- Automatically assigns IP addresses to devices  

---

## 🔁 NAT (Network Address Translation)

- Converts private IP → public IP  
- Used in routers for internet access  

---

## 💡 Hacker Perspective

Understanding IP, DNS, and NAT helps in:
- Identifying target systems  
- Mapping network structure  

---

# 🧱 OSI Model (Very Important)

## 7 Layers:

1. Physical  
2. Data Link  
3. Network  
4. Transport  
5. Session  
6. Presentation  
7. Application  

---

## 🌐 TCP/IP Model

- Network Interface  
- Internet  
- Transport  
- Application  

---

## 🔥 Key Concepts

### 🔹 TCP vs UDP
- TCP → reliable, connection-based  
- UDP → fast, no guarantee  

### 🔹 Ports
- 80 → HTTP  
- 443 → HTTPS  
- 22 → SSH  
- 21 → FTP  

### 🔹 3-Way Handshake
1. SYN  
2. SYN-ACK  
3. ACK  

---

## 💡 Hacker Perspective
