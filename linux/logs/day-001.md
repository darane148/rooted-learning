## Day 001 – 2026-04-04

**Focus:** Linux file system navigation & permissions
**Environment:** Kali Linux 2024.1 (local VM)

---

### Commands Run

```bash
# Navigate to root of filesystem and explore
ls -la /

# Understand what's in key directories
ls /etc | head -20
ls /var/log
ls /proc | head -10

# Find files owned by root
find / -user root -type f 2>/dev/null | head -20

# Check permissions on sensitive files
stat /etc/passwd
stat /etc/shadow
ls -la /etc/shadow

# View passwd file structure
cat /etc/passwd | head -5

# See who I am and what groups I'm in
whoami
id
groups

# Check file permissions with octal output
stat -c "%a %n" /etc/passwd
stat -c "%a %n" /etc/shadow

# Change permissions on a test file
touch /tmp/testfile
chmod 755 /tmp/testfile
ls -la /tmp/testfile
chmod 600 /tmp/testfile
ls -la /tmp/testfile

# Find all files with permissions 777
find /tmp -perm 777 2>/dev/null
```

### What Happened

```
$ stat /etc/shadow
  File: /etc/shadow
  Size: 1234
Access: (0640/-rw-r-----)  Uid: (    0/    root)   Gid: (   42/  shadow)

$ ls -la /etc/shadow
-rw-r----- 1 root shadow 1234 Apr  4 2026 /etc/shadow

$ id
uid=1000(kali) gid=1000(kali) groups=1000(kali),4(adm),24(cdrom),27(sudo)
```

The shadow file is only readable by root and the `shadow` group. My user `kali` is in the `sudo` group but NOT the `shadow` group, so I can't read it directly.

---

### Mistakes Made

1. **Ran `ls /proc` without `| head`** — it dumped thousands of entries. Always pipe to `head` when you don't know the output size.
2. **Forgot `2>/dev/null` on `find` at first** — the terminal got flooded with "Permission denied" errors. Redirecting stderr to null is standard practice for `find`.
3. **Confused `chmod 755` with `rwxr-xr-x`** — had to convert manually. Learned the trick: 4=read, 2=write, 1=execute. Add them up per group.

---

### What I Learned

- `/etc/passwd` is world-readable (everyone can read it) — it stores usernames and user info but NOT passwords anymore.
- `/etc/shadow` stores actual password hashes and is heavily restricted.
- Permission notation: `rwxr-xr-x` = owner can read/write/execute, group can read/execute, others can read/execute.
- `2>/dev/null` is essential when using `find` on the whole filesystem — there will be many permission-denied errors.
- `stat` gives more detail than `ls -la` — includes inode, blocks, access/modify/change times.
