## Day 002 тАУ 2026-04-04

**Focus:** Packet capture with tcpdump & Wireshark analysis
**Environment:** Kali Linux 2024.1, monitoring traffic on local network (eth0)

---

### Commands Run

```bash
# List available network interfaces
ip a
ip link show

# Capture all traffic on eth0 (write to file)
sudo tcpdump -i eth0 -w capture.pcap

# Capture with timestamps, verbose output
sudo tcpdump -i eth0 -v -tttt

# Filter: capture only HTTP traffic (port 80)
sudo tcpdump -i eth0 port 80 -A

# Filter: capture only traffic to/from specific host
sudo tcpdump -i eth0 host 192.168.56.101

# Capture ICMP packets only (ping traffic)
sudo tcpdump -i eth0 icmp

# Capture SYN packets (spot port scans)
sudo tcpdump -i eth0 'tcp[tcpflags] & tcp-syn != 0 and tcp[tcpflags] & tcp-ack == 0'

# Read a previously captured file
tcpdump -r capture.pcap

# Read and filter a pcap
tcpdump -r capture.pcap host 192.168.56.101 and port 80

# Count packets in capture
tcpdump -r capture.pcap | wc -l

# Open in Wireshark
wireshark capture.pcap &
```

---

### Mistakes Made

1. **Captured without `-w` first** тАФ the output scrolled by so fast it was unreadable. Always write to a file when doing serious capture, then analyze offline.
2. **Left `tcpdump` running on a busy interface** тАФ the pcap grew to 2GB in 5 minutes. Learned about `-c` flag to limit packet count: `tcpdump -i eth0 -c 1000 -w output.pcap`.
3. **Didn't understand the TCP flags** тАФ `[S]` = SYN, `[S.]` = SYN-ACK, `[.]` = ACK, `[P.]` = PSH-ACK (data), `[F.]` = FIN-ACK. Took a while to read the notation.

---

### What I Learned

- **The 3-way handshake** in action: SYN тЖТ SYN-ACK тЖТ ACK. Every TCP connection starts this way. In a port scan, SYN scans abort after SYN-ACK without completing the handshake.
- **tcpdump BPF filters** (Berkeley Packet Filter): extremely powerful syntax for filtering exactly what you want. `host`, `port`, `net`, `src`, `dst`, `and`, `or` тАФ these compose into complex filters.
- **`-A` flag** shows ASCII тАФ great for unencrypted protocols. **`-X` flag** shows hex + ASCII тАФ better for binary protocols.
- **HTTPS traffic looks like garbage** in tcpdump because it's TLS-encrypted. You'd need to perform a MITM or have the private key to decrypt it.
- **Wireshark** can follow TCP streams (Right-click тЖТ Follow тЖТ TCP Stream) тАФ shows the full conversation in a readable format.
